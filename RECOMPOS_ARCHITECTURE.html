What is RecompOS?

RecompOS is a progressive web application (PWA) for body recomposition — it helps you track your weight, body fat percentage, workouts, meals, and daily habits. Everything syncs in real time across all your devices (phone, tablet, laptop) with no app store installation needed.

It works offline first: data saves locally, then syncs to the cloud when you're back online. Changes propagate to your other devices in under 100 milliseconds via Supabase Realtime.

How it works, simply

Tech Stack

Built with modern, open-source tools that work together seamlessly:

⚡

SvelteKit

Modern framework with first-class service worker support — 42KB baseline, static adapter for GitHub Pages.

🗄️

Dexie.js

Wraps IndexedDB for offline. All data stored locally first, syncs to cloud when online.

☁️

Supabase

PostgreSQL database, anonymous auth, real-time WebSocket subscriptions, and serverless edge functions.

🧠

Gemini API

Google Gemini 2.5 Flash for AI body fat estimation from photos — 1,500 free requests/day.

🔔

Push API

Web Push notifications via VAPID keys. Alarms fire even when the app is closed.

🐙

GitHub Pages

Automated deploy via GitHub Actions on every push. Custom domain at recomp.nik.lat.

📱 Pages

Your app has six main screens, accessible from the bottom navigation bar:

Today — Dashboard

Your home base. See today's schedule at a glance, check your latest stats (weight lost, current weight, weeks to goal), and quickly log weight, calories, or steps.

Alarms — Smart Reminders

Alarms sync across all your devices via Supabase Realtime in under 100ms. Set repeating reminders with day-of-week selectors, toggle on/off, and they fire as push notifications even when the app is closed.

Recipes — Meal Plans

A collection of chicken curry recipes with both stovetop and Instant Pot instructions. Each recipe includes macros, ingredients, and embedded YouTube video tutorials.

Workouts — Exercise Library

Two structured workout sessions (A and B) targeting different muscle groups. Each exercise shows target area, rep scheme, and an embedded demonstration video.

Track — Progress Tracking

Log your weight daily and watch the trend on an SVG line chart. Track steps, monitor water intake (8-glass goal), and see your progress toward the 97 kg target.

Plan — Body Composition & Goals

Enter body measurements (height, neck, waist, hip) for a Navy-method body fat calculation. See projected goal weights for Lean/Fit/Healthy scenarios. Upload a photo for AI-powered body fat estimation via Gemini Vision.

🔄 How Data Flows

Every action you take follows the same path:

1

You log data — type a weight, tap a water drop, save a meal plan. The data is written immediately to IndexedDB (your device's local database), so it's saved even if you're offline.

2

Sync engine picks up the change and sends it to Supabase (the cloud database). The sync status dot at the top of the screen turns amber while syncing, green when done.

3

Supabase Realtime broadcasts the change to every other device that's signed into your account. Your phone, tablet, and laptop all update simultaneously.

4

Each device's sync engine receives the broadcast and updates its local IndexedDB. Your UI reacts automatically and shows the new data.

Key design decisions

📴

Offline-First

You can log data with no internet. It syncs when connectivity returns.

⚡

Real-Time

Changes appear on all devices in <100ms via WebSocket.

🔔

Push Alarms

Alarms fire as push notifications even when the app is closed.

🔐 Security & Privacy

Your data is private and secure:

  • Anonymous authentication — no email or password needed. A unique, anonymous identity is created when you first open the app.
  • Row-Level Security — Supabase policies ensure you can only see and edit your own data.
  • Publishable keys — the app uses restricted API keys that can only read/write your own data.
  • Photo analysis — body photos sent for AI estimation go directly to Gemini via a Supabase edge function. Your API key stays server-side.
